# KBO AI 데이터 서비스 — 프로젝트 기획 정리

## 1. 프로젝트 배경

### 왜 이 프로젝트를 하는가
- AI 기술을 실전 프로젝트에 적용하고 싶다
- 야구 관련 프로젝트를 하고 싶다
- SDD(Spec-Driven Development) 프로세스로 체계적으로, 실제 회사처럼 진행하고 싶다
- 최종적으로 개발까지 완성하는 것이 목표

### 개발자 현재 상황
- 개발 경험은 있으나 Python은 최근에 안 써서 감 잡기 필요
- 개발 환경은 다 갖춰져 있음
- GPU: GTX 1660 (6GB VRAM) — 노트북
- **비용 제약: 현재 단계에서 돈이 들면 안 됨 (LLM API 비용 포함)**

## 2. 시장 조사 결과 요약

### KBO 시장 현황
- 2024년 1,088만 → 2025년 **1,231만 관중** (역대 최다)
- 경기당 평균 17,100명, 331경기 매진(46%)
- 여성 관람객 비율 **55.5%**로 남성 역전 (2016년 42.9% → 2024년 55.5%)
- 20~30대 여성 팬이 성장 핵심 동력 (K팝 덕질 문화의 스포츠 이식)
- 경제 효과: 연간 생산유발액 약 **1조 1,121억 원**
- 중계권: 연 220억(2019~2023) → 연 **900억(2027~2031)** 4배 증가

### KBO 앱 생태계의 문제
- KBO 공식 앱 평점 **2.3/5** — "로고만 뜨고 앱이 안 켜진다", "올스타 투표 외에 쓸 일 없다"
- KBO STATS — "광고 끄자마자 강제종료 무한반복", "19금 웹툰 같은 이상한 광고"
- 프로야구 LIVE — "실시간 득점 알림이 안 뜬다"
- **MLB Ballpark 같은 통합 팬 앱이 KBO에는 없음**
- 모든 서비스가 파편화 (예매: 구단마다 다른 플랫폼, 기록: 네이버/KBO STATS/STATIZ 왔다갔다)

### STATIZ 신뢰도 하락
- 스포키 흡수 후 2025년 대규모 개편 → 중요 스탯 삭제, WAR 계산식 수정, 오류 다수
- 야구 덕후들이 **KBReport로 이주 중**
- 이 시점이 바로 더 나은 데이터 서비스로 진입할 타이밍

### 경쟁 서비스 비교 — 우리의 빈틈

| 항목 | KBO 공식 | STATIZ | KBReport | **우리 서비스** |
|-----|---------|--------|----------|--------------|
| 클래식 스탯 | O | O | O | O |
| 세이버메트릭스 | X | O (신뢰 하락) | O | O (자체 계산) |
| 스플릿 (vs좌/우, 득점권) | X | O | 일부 | O |
| 시즌 추이 그래프 | X | O | X | O |
| 투타 매치업 | X | 일부 | X | O |
| **복합 조건 질의** | **X** | **제한적** | **X** | **핵심 기능** |
| UX | 올드 | 복잡 | 깔끔 | 깔끔 + 그래프 |

→ KBO는 클래식만, STATIZ는 세이버 있지만 복합 조건 안 됨, KBReport도 고정된 테이블만. 드롭다운 5단 조합으로 자유롭게 질의하는 건 세 곳 모두 못 함. 이것이 핵심 차별점.

### NPB 비교 (참고)
- NPB 2024년 2,668만 관중 (KBO의 2배 이상 규모)
- 하지만 KBO가 사업적으로 더 매력적 — 성장률 폭발적, 디지털 서비스 생태계 미성숙
- NPB는 구단별 앱/서비스가 상대적으로 잘 갖춰져 있음
- KBO는 빈틈이 크기 때문에 소규모 팀도 진입 가능

## 3. 핵심 피드백 (외부 검증)

### 친구 피드백 — "올인원은 위험"
1. **네이버를 이길 수 없다** — 일정/라인업/스코어는 네이버가 이미 완벽하게 함
2. **데이터를 어디서 가져오나** — KBO 공식 API 없음, 스크래핑은 차단 리스크
3. **범위가 넓으면 완성도가 떨어진다** — 1인 개발로 올인원은 이도저도 아닌 앱이 됨
4. **챗봇이 핵심이 되면 안 됨** — "김도영 OPS"는 네이버 3초면 나옴
5. **네이버가 못 하는 것을 해야 함** — 복합 조건 질의가 진짜 가치

### 친구가 원하는 기능
1. 팀별 인스타 + 선수 인스타 찾기
2. 일정표
3. 이기는 승률
4. 기대되는 선수
5. 선발뿐 아니라 불펜/대타/대주자 한번에 보기

### 앱 리뷰 분석 결과 — 팬들이 진짜 원하는 것
1. **기본이 되는 앱** (앱이 켜지는 것, 광고 과하지 않은 것, 로그인 되는 것)
2. **소파스코어 수준의 실시간 알림**
3. **광고 없는 깔끔한 기록 앱**
4. **교체 선수 포함 전체 라인업**
5. **스포일러 방지 모드**
6. **통합 접근 — 여러 앱/사이트 왔다갔다 안 해도 되는 것**
7. **MLB 앱 수준의 품질**

## 4. 프로젝트 정의

### 한 줄 정의
**"네이버가 못 하는 KBO 복합 조건 질의를 드롭다운 조합으로 해결하는 데이터 대시보드"**
— STATIZ보다 깔끔하고, 네이버보다 깊게

### 핵심 차별화 (네이버가 못 하는 것)
- "2024년 좌타자 상대 FIP가 가장 낮은 투수 5명은?"
- "김도영이 주자 득점권에서 OPS가 얼마나 달라져?"
- "우리 팀이 5점 이상 뒤진 경기에서 역전한 확률은?"
- "금요일 야간경기 타율 1위는 누구?"
- 재미 요소: 팀스탯 종합 순위, 핫/콜드 선수, 이색 통계

### 형태
- **메인: 웹 대시보드** (LLM 비용 0원)
- 드롭다운 조합으로 복합 조건 질의 → 결과 + 그래프 시각화
- 향후 확장: 로컬 SLM 자연어 질의 기능 (실험적 — 아래 참고)

### 타겟 유저
- **1차 타겟: 데이터 야구팬** — STATIZ/KBReport 쓰는 세이버메트릭스 관심층. 복합 조건 질의 수요가 명확한 핵심 유저
- **2차 타겟: 중급 팬** — "야구 좀 안다"는 팬. 기록 비교, 선수 스플릿 조회, 커뮤니티 활동
- **3차 타겟 (향후): 커뮤니티 유저** — 팀별 게시판, 경기 중 소통. 커뮤니티 기능은 3순위이므로 여기 대응
- **아닌 것: 야구 입문자** — 네이버 검색으로 충분한 층. 이 서비스의 타겟이 아님

### ① 데이터 탐색기 (1순위 MVP — 핵심 차별화)

**드롭다운 5단 구성:**
```
[대상] → [조건] → [지표] → [정렬] → [범위]
```

**대상:** 타자 / 투수(선발/불펜) / 팀 / 특정 선수(검색)

**조건 필터:**
| 카테고리 | 세부 조건 |
|---------|----------|
| 상대 타자/투수 | vs 좌타, vs 우타, vs 좌투, vs 우투 |
| 주자 상황 | 주자 없음, 득점권, 만루 |
| 이닝 | 1~3회, 4~6회, 7~9회, 연장 |
| 홈/원정 | 홈, 원정 |
| 요일/시간 | 주중, 주말, 야간, 주간 |
| 점수차 | 리드, 동점, 비하인드, 5점차 이상 |
| 상대팀 | 특정 팀 선택 |

**지표 (풀 세이버메트릭스):**
| 카테고리 | 타자 | 투수 | 팀 |
|---------|------|------|-----|
| 클래식 | 타율, 홈런, 타점, 안타, 도루 | 승, 패, ERA, WHIP, 탈삼진 | 승률, 득점, 실점 |
| 세이버 | OPS, wOBA, wRC+, WAR, BABIP, ISO, BB%, K% | FIP, xFIP, K/9, BB/9, WAR, BABIP, LOB% | 팀 WAR, 팀 FIP, DRS |
| 상황별 | 득점권 OPS, 클러치 | 홀드율, 블론세이브 | 역전 승률, 접전 승률 |

**정렬:** 높은순 / 낮은순
**범위:** 상위 5명 / 10명 / 20명 / 전체
**데이터 범위:** 추후 결정 (데이터 수집 파이프라인 구축 후)

**결과 표시:**
- 메인: 순위 테이블 (순위, 선수명, 팀, 선택한 지표 + 보조 지표 2~3개)
- 보조: 막대 그래프 (Chart.js) — 선택한 지표 시각화
- 클릭 시: 해당 선수의 ② 세부정보 페이지로 이동

**기술 동작:**
유저 드롭다운 선택 → React가 조건을 JSON으로 조합 → FastAPI 백엔드로 요청 → 동적 SQL 빌더가 WHERE절 생성 → DB 조회 → JSON 응답 → 프론트에서 테이블 + 차트 렌더링

---

### ② 선수 세부정보 (1순위 MVP)

**프로필 헤더:** 선수명, 팀, 포지션, 등번호, 생년월일, 신체 정보, SNS 링크 아이콘(인스타/유튜브)
(사진은 저작권 이슈로 이니셜 아바타)

**MVP 3탭:**
| 탭 | 내용 |
|---|------|
| 클래식 | 타율/홈런/타점/안타/도루 (타자) 또는 승/패/ERA/WHIP/탈삼진 (투수) |
| 세이버메트릭스 | OPS, wOBA, wRC+, WAR, FIP, xFIP, BABIP 등 풀 지표 |
| 스플릿 | vs 좌/우, 득점권, 홈/원정, 이닝별, 월별 성적 분리 |

**후순위 탭 (데이터 확보 후):**
| 탭 | 내용 | 조건 |
|---|------|------|
| 추이 | 시즌별 주요 지표 변화 라인 그래프 | 멀티시즌 데이터 확보 후 |
| 매치업 | 특정 투수/타자 상대 성적 (투타 매치업) | 경기별 대진 데이터 파싱 필요 |

### ③ 순위 (1순위 MVP)

| 요소 | 내용 |
|-----|------|
| 팀 순위표 | 승/패/무/승률/게임차/연속 + 최근 5경기 흐름 |
| 팀스탯 비교 | 공격력(wRC+), 투수력(팀 FIP), 수비력(DRS), 주루(도루 성공률) 카드 |
| 선수 순위 | 타율/홈런/타점/도루/ERA 등 주요 지표별 TOP5~10 |

차별화: KBO 공식은 승/패/승률만 보여줌. 팀별 세이버메트릭스 비교는 우리만 함.

### ④ 일정/결과 (2순위)

| 요소 | 내용 |
|-----|------|
| 오늘 경기 | 대진, 시간, 구장, 선발투수 |
| 선발투수 프리뷰 | 양팀 선발투수의 최근 성적 + 상대팀 전적 (FIP/xFIP 포함) |
| 경기 결과 | 스코어, 이닝별 득점, 승리/패전/세이브 투수 |
| 캘린더 뷰 | 월간 일정 한눈에 (날짜 클릭 → 해당일 경기) |

차별화: 일정/결과 자체가 목적이 아니라 탐색기와 선수정보로 가는 진입점 역할. 선수 클릭 → ② 세부정보, "이 경기 데이터 탐색" → ① 탐색기로 연결.

### ⑤ 풀 라인업 (2순위)

| 구분 | 내용 |
|-----|------|
| 선발 라인업 | 1~9번 타순, 포지션, 주요 스탯 (타율/OPS 또는 ERA) |
| 선발 투수 | 오늘 선발의 시즌 성적 + 상대팀 전적 |
| 불펜 대기 | 셋업/마무리/롱릴리프 구분, ERA/홀드/세이브 |
| 벤치 | 대타/대주자 후보, 최근 대타 성적 |

차별화: 네이버는 선발 9명만 보여줌. 우리는 전체 엔트리를 한 화면에 역할별 구분. 불펜 투수별 연투 상황 표시. 각 선수 클릭 → ② 세부정보 이동.

---

### ⑥ 기록 조회 (2순위)

| 구분 | 내용 |
|-----|------|
| 선수별 | 특정 선수의 시즌 기록 전체 (클래식 + 세이버) |
| 팀별 | 팀 전체 타자/투수 기록 테이블 |
| 시즌별 | 연도 선택 → 해당 시즌 전체 기록 |

②와의 차이: ②는 한 선수를 깊게(줌인), ⑥은 여러 선수를 한 테이블에서 넓게(줌아웃).
차별화: KBO 공식은 클래식만. 우리는 세이버메트릭스 컬럼 토글 가능. 테이블 헤더 클릭으로 정렬.

### ⑦ 이색 통계 (3순위)

**미리 만들어둔 카드 형태 — 탐색기의 쇼케이스 역할:**

| 카테고리 | 예시 |
|---------|------|
| 요일/시간 | 금요일 야간경기 타율 TOP5, 주말 홈런왕 |
| 날씨/환경 | 비 온 날 승률 팀 순위, 돔 vs 야외 성적 차이 |
| 상황별 | 5점 이상 뒤진 역전 확률 팀 순위, 연장전 승률 |
| 이닝별 | 1회 선취점 팀 최종 승률, 9회 역전 횟수 |
| 기록 마일스톤 | "OO 선수 통산 100홈런까지 N개 남음" |
| 이상한 기록 | 몸에 맞는 공 가장 많이 맞은 선수, 삼진 안 당하는 타자 |

SNS 공유하기 좋은 포맷으로 디자인. 시즌 중 주기적으로 새 카드 추가.
유저 흐름: 이색 통계 카드 보고 → "나도 다른 조건으로 해볼래" → ① 탐색기로 이동.

### ⑧ 핫/콜드 선수 (3순위)

| 구분 | 내용 |
|-----|------|
| HOT 타자 | 최근 10경기 성적 급상승 (타율/OPS 기준) TOP5 |
| HOT 투수 | 최근 5경기 성적 급상승 (ERA/FIP 기준) TOP5 |
| COLD 타자 | 최근 10경기 성적 급하락 TOP5 |
| COLD 투수 | 최근 5경기 성적 급하락 TOP5 |

판단 기준: HOT/COLD 점수 = 최근 N경기 지표 - 시즌 전체 지표
표시: HOT 뱃지(빨강) / COLD 뱃지(파랑) + 시즌 vs 최근 성적 나란히 + 상승/하락 화살표.
매일 자동 업데이트 (배치 잡). ⑦ 이색 통계 페이지 안에 섹션으로 넣거나 홈 대시보드 위젯으로 배치.

- kbo-data — pip install kbodata, Chrome driver 기반. 스케줄/경기/타자/투수 DataFrame 변환
- KBO-Baseball-Data-Collection — 경기별 상세 데이터 체계적 크롤링

**서비스/설계 참고:**
- baseball_community — ERD(선수/팀/경기/기록 테이블 관계), API 엔드포인트 구조(선수 정보·팀 순위·일정/결과·라인업), 페이지 구성, 배포 아키텍처 참고

**ML/예측 모델 계열:**
- predict_kbo — 최근 10년 팀 스탯 기반 Logistic Regression으로 우승/포스트시즌 진출 확률 예측. Python, sklearn, Pandas, Seaborn. 향후 "승부 예측" 기능 참고
- Predict-WAR-with-Classic-Stats — 클래식 스탯(타율, 홈런, 타점 등)만으로 WAR를 예측하는 ML 모델. STATIZ WAR 신뢰도 하락 상황에서 자체 WAR 추정 기능 구현 시 핵심 참고
- baseball_ops_predict — STATIZ 크롤링 + 타자 OPS 예측 모델 (MAE 0.09). 파생변수 생성(행운의 안타, 타자 속도 계수 등) 방법론 참고

**이미지/비전 계열:**
- KBO_Baseball_image_analytics — 야구 이미지 분석. 현재 MVP에는 불필요하나, 향후 확장 시 참고. 교재 Ch6(컴퓨터 비전) 연계 학습용

**MLB 벤치마크:**
- pybaseball — Baseball Reference, FanGraphs, Statcast 데이터 스크래핑. 우리 프로젝트의 궁극적 목표가 "pybaseball의 KBO 버전"

| 리스크 | 대응 |

## 11. 개발 프로세스 & 타임라인

### SDD + 5단계 프로세스 (상세는 섹션 6 참고)

```
steering (프로젝트 기본 방침) ← 현재 단계
    ↓
requirements (요구사항 정리)
    ↓
design (설계)
    ↓
tasks (작업 분해)
    ↓
implementation (AI 에이전트로 구현)
    ↓
반복 (피드백 → 스펙 수정 → 재구현)
```

### 예상 타임라인 (에이전트 활용 기준)

| 단계 | 기간 | 누적 | 비고 |
|-----|------|------|------|
| SDD 문서 (steering → requirements → design) | 1~2주 | 1~2주 | 지금부터 |
| **MVP (①②③ + 데이터 파이프라인)** | 4~6주 | 6~8주 | 데이터 수집이 병목 |
| 2순위 (④⑤⑥) | 1.5~2주 | 8~10주 | CRUD 위주, 에이전트 강점 |
| 3순위 — 커뮤니티 제외 (⑦⑧⑪⑫) | 1.5~2주 | 10~12주 | 개발 부담 낮음 |
| 3순위 — 커뮤니티 + 직관로그 (⑨⑩) | 3~4주 | 13~16주 | 로그인/게시판 풀세트 |

**총 예상: 약 3~4개월**

에이전트가 빠르게 해주는 것: API 엔드포인트, React 컴포넌트, SQL 쿼리, CRUD, 차트 렌더링 (코딩 시간 60~70% 절감)
에이전트가 못 해주는 것 (진짜 시간 먹는 곳): 데이터 수집 파이프라인 안정화, 세이버메트릭스 정확도 크로스체크, 배포 후 실사용 버그, 데이터 입력 노가다 (응원가 가사, SNS URL 등)
