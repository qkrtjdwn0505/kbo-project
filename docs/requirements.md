> 상위 문서: steering.md
> 이 문서는 15개 기능 각각의 상세 요구사항을 정의합니다.
> 각 요구사항에는 Acceptance Criteria(완료 조건)가 포함됩니다.
> AI 에이전트에게 구현을 지시할 때 이 문서를 @file로 참조합니다.
> 최종 수정일: 2026-03-16

## 공통 요구사항

### UI/UX 공통
- 반응형 웹 (데스크톱 우선, 모바일 대응)
- 다크모드 미지원 (MVP) — 라이트 테마만
- 한국어 전용
- 모든 선수 이름 클릭 → ② 선수 세부정보 페이지로 이동
- 모든 팀 이름 클릭 → 해당 팀 기록 페이지로 이동
- 로딩 상태 표시 (스켈레톤 UI 또는 스피너)
- 에러 상태 표시 (데이터 조회 실패 시 안내 메시지)

### 데이터 공통
- 세이버메트릭스 지표는 소수점 3자리까지 표시 (타율 .328, OPS 1.010)
- 승률은 할푼리 표기 (.613)
- 숫자는 천 단위 구분 (1,231만)
- 데이터 출처 표기 (페이지 하단에 "데이터 출처: KBO, KBReport")
- 규정 이닝/규정 타석 미달 선수는 별도 표시

### 네비게이션
- 상단 네비게이션 바: 홈, 탐색기, 선수, 팀, 일정, 기록, 이색통계
- 현재 페이지 활성 표시
- 로고 클릭 → 홈으로 이동

## 1순위 MVP

---

### REQ-01: 데이터 탐색기

**설명:** 드롭다운 5단 조합으로 KBO 복합 조건 질의를 실행하고 결과를 테이블 + 차트로 보여주는 핵심 기능

**유저 스토리:**
- 데이터 야구팬으로서, "좌타자 상대 FIP가 가장 낮은 투수 5명"을 드롭다운 조합만으로 바로 볼 수 있다
- 결과를 보고 특정 선수를 클릭하면 해당 선수의 세부정보를 바로 확인할 수 있다

**드롭다운 구성:**

| 단계 | 이름 | 값 |
|-----|------|-----|
| 1단 | 대상 | 타자 / 투수(전체) / 투수(선발) / 투수(불펜) / 팀 |
| 2단 | 조건 | 전체 / vs좌타 / vs우타 / vs좌투 / vs우투 / 득점권 / 만루 / 주자없음 / 1~3회 / 4~6회 / 7~9회 / 홈 / 원정 / 주중 / 주말 / 야간 / 주간 / 리드 / 동점 / 비하인드 / 상대팀선택 |
| 3단 | 지표 | (대상에 따라 동적 변경 — 아래 참고) |
| 4단 | 정렬 | 높은순 / 낮은순 |
| 5단 | 범위 | 상위 5명 / 10명 / 20명 / 전체 |

**3단 지표 — 대상별:**

타자 선택 시: 타율, 홈런, 타점, 안타, 도루, OPS, wOBA, wRC+, WAR, BABIP, ISO, BB%, K%, 득점권OPS

투수 선택 시: ERA, 승, 패, WHIP, 탈삼진, FIP, xFIP, K/9, BB/9, WAR, BABIP, LOB%, 홀드, 세이브, 블론세이브

팀 선택 시: 승률, 득점, 실점, 팀WAR, 팀FIP, 팀wRC+, DRS합계, 역전승률, 접전승률

**결과 표시:**
- 순위 테이블: 순위, 선수명(클릭 가능), 팀, 선택한 지표 값, 보조 지표 2~3개
- 막대 그래프: Chart.js 수평/수직 막대 — 선택한 지표 시각화
- 현재 질의 요약 표시: "타자 · 득점권 · OPS · 높은순 · 상위 5명"

**Acceptance Criteria:**
1. 5개 드롭다운을 모두 선택하면 결과가 2초 이내에 표시된다
2. 대상 변경 시 3단(지표) 드롭다운이 동적으로 변경된다
3. 조건을 변경할 때마다 결과가 자동 갱신된다
4. 결과 테이블의 선수 이름 클릭 → REQ-02 선수 세부정보로 이동한다
5. 결과가 없는 조합일 때 "해당 조건의 데이터가 없습니다" 메시지를 표시한다
6. 막대 그래프가 결과와 동기화되어 표시된다
7. 규정 타석/이닝 미달 선수는 별도 표시(회색 또는 주석)된다

### REQ-02: 선수 세부정보

**설명:** 특정 선수의 기록을 클래식/세이버메트릭스/스플릿 3개 탭으로 깊게 보여주는 페이지

**유저 스토리:**
- 데이터 야구팬으로서, 김도영의 세이버메트릭스와 상황별 스플릿을 한 페이지에서 확인할 수 있다
- 탐색기에서 클릭해서 왔든 직접 검색했든 같은 페이지를 본다

**프로필 헤더:**
- 선수명, 팀명, 포지션, 등번호, 생년월일, 신체 정보 (키/몸무게)
- 이니셜 아바타 (사진은 저작권 이슈로 미포함)
- SNS 링크 아이콘 (인스타/유튜브 — 데이터 있는 경우만 표시)

**탭 1 — 클래식:**

타자: 경기, 타석, 타수, 안타, 2루타, 3루타, 홈런, 타점, 도루, 도실, 볼넷, 삼진, 타율, 출루율, 장타율, OPS

투수: 경기, 승, 패, 세이브, 홀드, 이닝, 피안타, 피홈런, 볼넷, 삼진, 자책점, ERA, WHIP

**탭 2 — 세이버메트릭스:**

타자: wOBA, wRC+, WAR, BABIP, ISO, BB%, K%, OPS+, 득점기여율

투수: FIP, xFIP, WAR, BABIP, LOB%, K/9, BB/9, HR/9, K/BB

**탭 3 — 스플릿:**
- vs 좌투/우투 성적 (타자 기준) 또는 vs 좌타/우타 성적 (투수 기준)
- 득점권/주자없음 성적 비교
- 홈/원정 성적 비교
- 이닝별 성적 (1~3, 4~6, 7~9)
- 월별 성적 (4월~10월)

**후순위 탭 (MVP 이후):**
- 추이: 시즌별 주요 지표 변화 라인 그래프
- 매치업: 특정 투수/타자 상대 성적

**선수 검색:**
- 페이지 상단 검색창
- 자동완성 (2글자 이상 입력 시 후보 표시)
- 팀 필터 옵션

**Acceptance Criteria:**
1. 선수 이름 검색 시 자동완성이 동작한다 (2글자 이상, 300ms 이내)
2. 3개 탭 전환이 페이지 새로고침 없이 동작한다
3. 클래식 탭의 모든 수치가 소수점 규칙에 맞게 표시된다
4. 세이버메트릭스 탭의 WAR, FIP 등이 KBReport 값과 ±0.1 이내로 일치한다
5. 스플릿 탭에서 vs좌/우, 홈/원정, 득점권 성적이 정확히 표시된다
6. SNS 링크가 있는 선수는 아이콘이 표시되고, 클릭 시 외부 링크로 이동한다
7. 존재하지 않는 선수 검색 시 "검색 결과가 없습니다" 메시지를 표시한다

### REQ-03: 순위

**설명:** 팀 순위표와 팀스탯 종합 비교를 보여주는 페이지

**유저 스토리:**
- 중급 팬으로서, 단순 승패뿐 아니라 "공격력 1위 팀, 투수력 1위 팀"을 한눈에 비교할 수 있다

**팀 순위표:**
- 순위, 팀명(클릭 가능), 승, 패, 무, 승률, 게임차, 연속(연승/연패), 최근5경기 흐름(승패 아이콘)
- 헤더 클릭 → 해당 컬럼 기준 정렬

**팀스탯 종합 비교 카드:**
- 공격력 1위 (wRC+ 기준)
- 투수력 1위 (팀 FIP 기준)
- 수비력 1위 (DRS 합계 기준)
- 주루 1위 (도루 성공률 기준)
- 각 카드에 팀명 + 수치 표시

**선수 순위:**
- 주요 지표별 TOP5: 타율, 홈런, 타점, 도루, ERA, 승, 탈삼진, 세이브
- 선수 이름 클릭 → REQ-02 이동

**Acceptance Criteria:**
1. 팀 순위표가 정규시즌 공식 순위와 일치한다
2. 최근 5경기 흐름이 정확히 표시된다
3. 팀스탯 종합 비교 카드 4개가 정확한 팀과 수치를 보여준다
4. 선수 순위 TOP5가 공식 기록과 일치한다
5. 모든 팀명, 선수명이 클릭 가능하고 올바른 페이지로 이동한다

### REQ-04: 일정/결과

**설명:** 오늘 경기 일정과 결과를 보여주며, 탐색기와 선수정보로의 진입점 역할

**유저 스토리:**
- 팬으로서, 오늘 경기의 선발투수와 간단한 프리뷰를 확인할 수 있다
- 경기 결과에서 궁금한 선수를 클릭해 세부정보로 바로 갈 수 있다

**오늘 경기:**
- 대진 (홈 vs 원정), 시간, 구장
- 선발투수 양팀 표시 (이름 + 시즌 ERA + 상대팀 전적)
- 경기 전: "18:30 예정" / 경기 후: 최종 스코어

**선발투수 프리뷰:**
- 양팀 선발투수의 최근 5경기 성적 (ERA, FIP, WHIP)
- 상대팀 통산 전적

**경기 결과:**
- 이닝별 득점 스코어보드
- 승리/패전/세이브 투수
- 주요 기록 (홈런, 멀티히트 등)

**캘린더 뷰:**
- 월간 캘린더에서 날짜 클릭 → 해당일 경기 목록

**Acceptance Criteria:**
1. 오늘 날짜의 경기 목록이 정확히 표시된다
2. 선발투수 정보가 경기 시작 전에 표시된다
3. 경기 종료 후 결과가 정확히 반영된다
4. 캘린더에서 날짜 클릭 시 해당일 경기가 표시된다
5. 모든 선수 이름 클릭 → REQ-02로 이동한다
6. 선발투수 프리뷰에 세이버메트릭스(FIP) 포함된다

### REQ-05: 풀 라인업

**설명:** 경기별 선발 + 불펜 + 벤치를 한 화면에서 역할별로 보여주는 기능

**유저 스토리:**
- 팬으로서, 오늘 경기에 누가 선발이고 불펜에 누가 대기 중인지 한눈에 확인할 수 있다

**선발 라인업:**
- 1~9번 타순, 이름(클릭 가능), 포지션, 주요 스탯 1개 (타자: 타율, 투수: ERA)

**선발 투수:**
- 오늘 선발의 시즌 성적 요약 (승/패, ERA, FIP, WHIP)

**불펜:**
- 셋업/마무리/롱릴리프 역할 구분 표시
- 각 투수: 이름, ERA, 홀드 또는 세이브
- 전날 등판 여부 표시 (연투 상황 확인)

**벤치:**
- 대타/대주자 후보 목록
- 최근 대타 성적 (있는 경우)

**Acceptance Criteria:**
1. 경기별 선발 라인업 9명이 정확히 표시된다
2. 선발 + 불펜 + 벤치가 한 화면에서 역할별로 구분된다
3. 불펜 투수의 전날 등판 여부가 표시된다
4. 모든 선수 이름 클릭 → REQ-02로 이동한다
5. 라인업이 아직 발표 안 된 경기는 "라인업 미발표" 표시

---

### REQ-06: 기록 조회

**설명:** 여러 선수를 한 테이블에서 넓게 비교하는 줌아웃 뷰 (REQ-02의 줌인과 대비)

**유저 스토리:**
- 데이터 팬으로서, KIA 전체 타자의 기록을 한 테이블에서 보고 원하는 지표로 정렬할 수 있다

**필터:**
- 팀 선택 (전체 / 특정 팀)
- 포지션 선택 (전체 / 투수 / 포수 / 내야수 / 외야수)
- 시즌 선택 (연도)

**테이블:**
- 기본 컬럼: 순위, 선수명, 팀, 포지션 + 클래식 스탯
- "세이버 보기" 토글 → 컬럼이 세이버메트릭스로 전환
- 헤더 클릭 → 해당 지표 기준 정렬
- 선수 이름 클릭 → REQ-02 이동

**Acceptance Criteria:**
1. 팀별/포지션별/시즌별 필터가 정확히 동작한다
2. "세이버 보기" 토글이 클래식 ↔ 세이버 컬럼을 전환한다
3. 헤더 클릭 정렬이 오름차순/내림차순으로 토글된다
4. 전체 선수를 불러와도 페이지가 느려지지 않는다 (페이지네이션 또는 가상 스크롤)
5. 선수 이름 클릭 → REQ-02로 이동한다

## 3순위

### REQ-07: 이색 통계

**설명:** 미리 계산된 재미있는 통계를 카드 형태로 보여주는 페이지 (탐색기의 쇼케이스)

**유저 스토리:**
- 중급 팬으로서, "금요일 야간경기 타율 TOP5" 같은 재미있는 통계를 보고 SNS에 공유할 수 있다

**카드 카테고리:**
- 요일/시간: 금요일 야간 타율 TOP5, 주말 홈런왕
- 날씨/환경: 비 온 날 승률 팀 순위, 돔 vs 야외 성적 차이
- 상황별: 5점 이상 뒤진 역전 확률 팀 순위, 연장전 승률
- 이닝별: 1회 선취점 팀 최종 승률, 9회 역전 횟수
- 기록 마일스톤: "OO 선수 통산 100홈런까지 N개 남음"
- 이상한 기록: 몸에 맞는 공 가장 많이 맞은 선수, 삼진 안 당하는 타자

**각 카드 구조:**
- 제목 (예: "금요일 야간경기 타율 TOP5")
- 순위 리스트 (순위, 선수명, 팀, 수치)
- "탐색기에서 더 보기" 링크 → REQ-01로 이동

**Acceptance Criteria:**
1. 최소 10개 이상의 이색 통계 카드가 표시된다
2. 각 카드의 데이터가 정확하다 (수동 크로스체크)
3. 카드 내 선수 이름 클릭 → REQ-02로 이동한다
4. "탐색기에서 더 보기" 클릭 → REQ-01로 해당 조건이 설정된 채 이동한다
5. 주기적으로 데이터가 업데이트된다 (배치 잡)

---

### REQ-08: 핫/콜드 선수

**설명:** 최근 성적이 급상승/급하락한 선수를 자동으로 감지해 보여주는 기능

**유저 스토리:**
- 팬으로서, 오늘 경기 전에 "요즘 누가 잘 치고 누가 부진한지" 빠르게 확인할 수 있다

**HOT/COLD 계산:**
```
점수 = 최근 N경기 지표 - 시즌 전체 지표
```
- HOT 타자: 최근 10경기 타율/OPS 기준 상위 5명
- HOT 투수: 최근 5경기 ERA/FIP 기준 상위 5명
- COLD 타자: 최근 10경기 기준 하위 5명
- COLD 투수: 최근 5경기 기준 하위 5명

**표시:**
- HOT 뱃지 (빨강) / COLD 뱃지 (파랑)
- 시즌 성적 vs 최근 성적 나란히 표시
- 상승/하락 화살표 + 차이값
- 선수 이름 클릭 → REQ-02 이동

**Acceptance Criteria:**
1. HOT/COLD 점수가 정확히 계산된다
2. 매일 자동 업데이트된다
3. HOT 5명, COLD 5명이 각각 타자/투수로 표시된다
4. 시즌 vs 최근 수치가 나란히 비교 표시된다

### REQ-09: 커뮤니티

**설명:** 데이터 서비스와 결합된 야구 팬 커뮤니티 (유저 확보 후 오픈)

**유저 스토리:**
- 데이터 팬으로서, 탐색기에서 찾은 결과를 게시글에 첨부해 다른 팬들과 토론할 수 있다

**게시판:**
- 팀별 게시판 (10개 구단)
- 자유 게시판
- 데이터 토론 게시판

**게시글:**
- 제목, 본문, 작성자, 작성일
- 이미지 첨부
- 데이터 카드 첨부 (탐색기/이색통계 결과를 원클릭으로 삽입)
- 댓글, 좋아요

**사전 조건:**
- 회원가입/로그인 시스템 필요
- 출시 전에 탐색기/이색통계의 "공유 링크 생성" 기능을 먼저 구현

**Acceptance Criteria:**
1. 회원가입/로그인이 동작한다
2. 게시글 CRUD(작성/읽기/수정/삭제)가 동작한다
3. 탐색기 결과를 게시글에 데이터 카드로 첨부할 수 있다
4. 팀별 게시판 필터링이 동작한다
5. 댓글 작성/삭제가 동작한다

### REQ-10: 직관로그

**설명:** 직관한 경기를 기록하고 개인 직관 통계를 보여주는 기능 (커뮤니티와 동시 출시)

**유저 스토리:**
- 팬으로서, "올해 15번 직관 가서 직관 승률 73%"를 확인하고 친구들에게 공유할 수 있다

**경기 기록:**
- "오늘 경기 추가" → 날짜/구장 선택 → DB에서 해당 경기 자동 매칭
- 한줄평 작성 (선택)
- 오늘의 MVP 선택 (선택)

**직관 통계 대시보드:**
- 총 직관 횟수
- 전체 직관 승률
- 팀별 직관 승률
- 구장별 방문 횟수
- 월별 분포 차트

**공유:**
- "나의 직관 리포트" 카드 이미지 자동 생성 → SNS 공유

**사전 조건:**
- REQ-09의 회원가입/로그인 시스템 공유

**Acceptance Criteria:**
1. 날짜/구장 선택 시 해당 경기가 자동 매칭된다
2. 직관 승률이 정확히 계산된다
3. 팀별/구장별 통계가 정확히 표시된다
4. 직관 리포트 카드 이미지가 생성된다

### REQ-11: 응원가 페이지

**설명:** 구단별/선수별 응원가 가사와 유튜브 영상을 모아보는 페이지

**유저 스토리:**
- 신규 팬으로서, 다음 직관 전에 우리 팀 응원가를 미리 배울 수 있다

**구조:**
- 구단별 탭 (10개 구단)
- 각 구단: 팀 공통 응원가 목록 + 선수별 응원가 목록
- 각 응원가: 가사 텍스트 + 유튜브 임베드 (있는 경우)
- 선수 이름 클릭 → REQ-02 이동

**데이터:**
- DB에 가사 텍스트 + 유튜브 URL 수동 입력
- 주요 선수 100명 + 팀 공통 10개 = 약 110개 항목

**Acceptance Criteria:**
1. 10개 구단 탭 전환이 동작한다
2. 각 구단의 팀 응원가 + 선수 응원가가 표시된다
3. 유튜브 임베드가 정상 재생된다
4. 선수 이름 클릭 → REQ-02로 이동한다
5. 가사 텍스트가 읽기 쉽게 줄바꿈 처리된다

### REQ-12: SNS 허브

**설명:** 구단별/선수별 SNS 링크를 모아보고, 유튜브 최신 영상을 임베드하는 페이지

**유저 스토리:**
- 팬으로서, 내가 응원하는 팀과 선수의 인스타/유튜브를 한곳에서 찾을 수 있다

**기존 기능 연동:**
- REQ-02 선수 세부정보 → 프로필 헤더에 SNS 아이콘 링크
- REQ-03 순위 → 팀명 옆에 구단 SNS 아이콘 링크

**별도 SNS 허브 페이지:**
- 구단별 탭 (10개 구단)
- 각 구단: 공식 인스타 링크 + 유튜브 최신 영상 3~5개 임베드
- 구단 소속 주요 선수 인스타 링크 목록

**데이터:**
- 구단 SNS URL: DB에 수동 입력 (10개)
- 선수 인스타 URL: DB에 수동 입력 (주요 50~100명)
- 유튜브 최신 영상: YouTube Data API로 자동 갱신

**Acceptance Criteria:**
1. 10개 구단 탭 전환이 동작한다
2. 각 구단의 공식 SNS 링크가 올바른 외부 페이지로 이동한다
3. 유튜브 최신 영상 3~5개가 임베드되어 재생 가능하다
4. 선수 인스타 링크가 올바르게 동작한다
5. YouTube Data API 무료 할당량 내에서 동작한다

### REQ-13: AI 자연어 질의 (실험적)

**설명:** 자연어로 KBO 데이터를 질의하는 챗봇 위젯 (드롭다운으로 못 만드는 자유로운 질문용)

**유저 스토리:**
- 데이터 팬으로서, "김도영이 삼성 상대로 야간경기에서 어떤 성적이야?" 같이 드롭다운으로 표현하기 어려운 질문을 자연어로 할 수 있다

**구조:**
- 대시보드 우측 하단 챗봇 위젯
- 자연어 입력 → 로컬 SLM이 SQL로 변환 → REQ-01과 같은 DB 조회 → 결과 표시
- 로컬 SLM: Phi-3-mini INT4 + Ollama (비용 0원)

**제약:**
- ⚠️ GTX 1660에서 성능 불안정 가능
- 원칙: 안 되면 드롭다운(REQ-01)으로 충분. 이 기능 없이도 서비스가 완전 동작해야 함

**Acceptance Criteria:**
1. 자연어 입력이 SQL로 변환되어 결과가 표시된다
2. 변환 실패 시 "이 질문은 탐색기에서 직접 검색해보세요" + 탐색기 링크를 표시한다
3. 응답 시간이 5초 이내이다
4. 이 기능이 꺼져 있어도 서비스의 모든 다른 기능이 정상 동작한다

### REQ-14: 승부 예측

**설명:** 오늘 경기의 승리 확률을 ML 모델로 예측하는 기능

**유저 스토리:**
- 팬으로서, 오늘 경기에서 "우리 팀이 이길 확률이 몇 %인지" 참고할 수 있다

**구조:**
- REQ-04 일정/결과 페이지에 통합
- 각 경기 옆에 "AI 예측: KIA 62% vs 삼성 38%" 표시
- 예측 근거 요약 (최근 5경기 성적, 선발투수 FIP, 홈/원정)

**모델:**
- sklearn Logistic Regression 또는 XGBoost (predict_kbo 참고)
- 입력: 최근 팀 성적 + 선발투수 스탯 + 홈/원정 + 상대전적
- CPU 학습으로 충분

**Acceptance Criteria:**
1. 오늘 경기의 승리 확률이 % 단위로 표시된다
2. 예측 근거가 간략히 표시된다
3. "이 예측은 참고용이며 실제 결과와 다를 수 있습니다" 면책 문구 포함
4. 시즌 진행에 따라 모델이 주기적으로 재학습된다

- 이색 통계 카드 → 인스타/트위터용 이미지 변환

### REQ-DATA-01: 데이터 수집 파이프라인

**설명:** KBO 경기/선수/팀 데이터를 수집하여 자체 DB에 저장하는 파이프라인

**수집 대상:**
- 경기별 스코어보드 (이닝별 득점)
- 경기별 박스스코어 (타자/투수 개인 기록)
- 선발 라인업
- 선수 프로필 (이름, 팀, 포지션, 생년월일, 등번호)
- 팀 순위

**수집 방법:**
1. kbo-data 라이브러리로 벌크 수집
2. 부족한 부분은 KBO 홈페이지 직접 크롤링 (requests + BeautifulSoup)
3. 세이버메트릭스 기초 데이터는 KBReport/STATIZ 참고

**Acceptance Criteria:**
1. 지정한 기간의 경기 데이터가 자동으로 수집된다
2. 수집된 데이터가 SQLite DB에 정상 저장된다
3. 중복 데이터가 발생하지 않는다 (upsert)
4. 크롤링 실패 시 에러 로그가 남고, 재시도 로직이 동작한다
5. rate limiting을 준수한다 (요청 간 1초 이상 간격)

### REQ-DATA-02: 세이버메트릭스 계산 엔진

**설명:** 수집된 기초 데이터로 세이버메트릭스 지표를 자체 계산

**계산 지표:**
- 타자: wOBA, wRC+, WAR, BABIP, ISO, BB%, K%
- 투수: FIP, xFIP, WAR, BABIP, LOB%, K/9, BB/9, HR/9
- 수비: DRS (가능한 경우)

**계산 원칙:**
- FanGraphs Library 공식 기준
- KBO 리그 상수 적용 (KBReport STAT Dic 참고)
- 모든 계산 함수에 단위 테스트 필수

**Acceptance Criteria:**
1. 모든 계산 함수에 단위 테스트가 작성되어 있다
2. 계산 결과가 KBReport 공개 수치와 ±0.1 이내로 일치한다 (WAR 기준)
3. 리그 상수가 시즌별로 올바르게 적용된다
4. 새 경기 데이터 추가 시 지표가 자동 재계산된다

### REQ-DATA-03: 배치 업데이트

**설명:** 시즌 중 주기적으로 새 경기 데이터를 수집하고 지표를 갱신

**Acceptance Criteria:**
1. 배치 스크립트가 매일 지정 시간에 실행된다
2. 새 경기 데이터만 추가 수집된다 (전체 재수집 아님)
3. 세이버메트릭스 지표가 새 데이터 반영하여 재계산된다
4. 이색 통계, 핫/콜드 선수가 갱신된다
5. 실행 로그가 남는다
